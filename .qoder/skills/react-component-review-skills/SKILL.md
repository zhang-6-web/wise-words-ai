# React 组件审查技能

## 描述
审查 React 组件的代码质量，包括性能优化、最佳实践和常见陷阱

## 使用场景
- 当用户说“审查我的 React 组件”
- 当用户问“这个组件写得怎么样”
- 当用户粘贴 React 代码并问“有什么问题吗”
- 在 Code Review 时自动触发

## 执行步骤

### 1. 检查基础结构
- [ ] 组件命名是否 PascalCase（如 `UserCard` 而非 `userCard`）
- [ ] 是否有默认导出或命名导出
- [ ] 组件文件是否放在正确目录

### 2. 性能优化检查
- [ ] 是否用了 `React.memo` 包装纯展示组件
- [ ] `useCallback` 和 `useMemo` 是否合理使用
- [ ] 依赖数组是否完整（`useEffect` 的依赖）
- [ ] 是否有不必要的重渲染

### 3. 代码规范检查
- [ ] Props 是否有类型定义（TypeScript 或 PropTypes）
- [ ] 是否解构 props（`const { name, age } = props`）
- [ ] 事件处理函数是否用 `handle` 前缀（如 `handleClick`）
- [ ] 条件渲染是否清晰（避免三元表达式嵌套过深）

### 4. 常见陷阱排查
- [ ] 是否在 `useEffect` 里更新 state 造成死循环
- [ ] 列表渲染是否有唯一的 `key` 属性
- [ ] 是否直接修改 props 或 state
- [ ] 异步操作是否有清理函数

## 输出格式
按照以下格式输出审查结果：

### ✅ 通过项
- 列举做得好的地方

### ⚠️ 需要改进
- 问题1：[问题描述] → [修改建议]
- 问题2：[问题描述] → [修改建议]

### 🔥 严重问题
- 问题描述 + 为什么严重 + 如何修复

## 注意事项
- 用户需要提供完整的组件代码
- 如果是 TypeScript 组件，审查会更准确
- 建议同时提供组件的使用场景（如表单、列表等）
